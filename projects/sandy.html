sandy サンディー

Sandy is a 3D renderer built from scratch in C, compiling direct to WebAssembly.

I place individual pixels on a framebuffer which is sent to Javascript for blitting to the web canvas

This project was mainly a way for me to refresh my skills in C++/C as it still seems so prevalent in the games industry.

I initially wrote it completely in C++, linking to my Rust Vulkan and WebGl engine.

After figuring out how to compile directly to WebAssembly I ended up switching to C, as I felt it fit the low-level, no_std style better.

The demo above is currently rendering the Spyro Artisans homeworld as a test

When I revisit this project I aim to programatically place and generate meshes

Coupling that with a tile map for easy level sketching I think I could create a cool top-down 3D rpg or something

UV mapped textures are working correctly (and did in the C++ version of this project), however my obj loader currently messes up the UVs
